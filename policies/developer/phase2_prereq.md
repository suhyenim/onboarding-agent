# 2단계 - 사전 준비 (AX 개발자)

## 이 단계는
AX 개발자가 AI 앱과 LLM(거대 언어 모델) 서비스를 만들기 전에, 코드를 다루는 가장 기본인 셸(터미널에 명령을 입력하면 받아서 실행해 주는 프로그램)과 계정 환경이 제대로 잡혀 있는지 확인하는 단계입니다. 여기가 흔들리면 이후 모든 빌드와 배포(만든 앱을 서버에 올려 실제로 돌리는 것)가 같이 흔들리므로, 가장 먼저 토대를 다집니다.

## 무엇을 준비하나
- 개발 계정: 사내 GitHub(코드를 저장하고 함께 작업하는 사이트) 조직 초대를 수락하고, 본인 계정으로 로그인이 되는지 확인합니다. 코드를 받고 올리려면 계정이 먼저 살아 있어야 합니다.
- 터미널 셸: macOS나 Linux에서 터미널을 엽니다. 모든 개발 명령은 이 셸에서 시작하므로, 셸이 정상으로 응답하는지부터 봅니다.
- 작업 폴더: 온보딩 코드를 둘 폴더를 만들고 그 안으로 들어갑니다.

```bash
# 셸이 응답하는지, 지금 어느 폴더에 있는지 확인
pwd

# 어떤 OS에서 작업 중인지 확인
uname -s

# 작업 폴더를 만들고 이동
mkdir -p ~/work/onboarding && cd ~/work/onboarding
```

## 검증 항목
- pwd: 작업 디렉토리에 접근할 수 있는지 (셸이 현재 경로를 반환)
- uname -s: OS 정보를 읽을 수 있는지 (실행 환경 확인)

## 문제 해결 (트러블슈팅)
- pwd가 동작하지 않으면 터미널 자체가 비정상입니다. 터미널을 새로 열거나 다시 로그인합니다.
- uname을 못 찾으면 기본 셸 도구가 빠진 환경입니다. macOS는 Command Line Tools(맥에서 개발 명령을 쓰게 해 주는 기본 도구 모음)를, Linux는 coreutils(리눅스 기본 명령어 묶음) 설치를 확인합니다.
- 계정 로그인이 막히면 사내 GitHub 조직 초대 메일을 다시 확인하고 관리자에게 재발송을 요청합니다.
